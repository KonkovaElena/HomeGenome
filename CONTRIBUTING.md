# Contributing

HomeGenome is a docs-first repository. Contributions are welcome when they improve technical accuracy, provenance, clarity or publication safety.

## Good First Contributions

- fix broken links, inconsistent tool versions or factual drift;
- improve bibliography quality and source provenance;
- tighten non-medical disclaimers and safety boundaries;
- improve diagrams, tables and repository navigation;
- propose focused additions to the reference architecture.

## Out Of Scope

- personal medical interpretation requests;
- uploads of personal genomes, VCF files, BAM files or any other private biological data;
- advice that looks like patient treatment guidance;
- vague hype edits that broaden claims without evidence.

Project-wide safety perimeter: [docs/reference/research-and-safety-boundary.md](docs/reference/research-and-safety-boundary.md).

## Contribution Rules

1. keep the diff focused;
2. separate fact from recommendation and forecast;
3. cite primary sources or official vendor documentation for new external claims;
4. do not add unverifiable numbers or benchmarks;
5. do not submit secrets, credentials or sensitive biological data;
6. if a change touches publication safety, review [docs/evidence/publication-readiness-audit-2026-04-21.md](docs/evidence/publication-readiness-audit-2026-04-21.md) first.

## Pull Request Expectations

Every pull request should explain:

1. what changed;
2. why the change was needed;
3. which sources were checked;
4. whether the change affects publication safety, legal provenance or medical framing.

## Before You Open An Issue Or PR

1. read [README.md](README.md);
2. check [docs/how-to/documentation-master-plan-2026-04-21.md](docs/how-to/documentation-master-plan-2026-04-21.md) if your change is structural;
3. use [SECURITY.md](SECURITY.md) instead of public issues for sensitive disclosures;
4. use [SUPPORT.md](SUPPORT.md) for support boundaries.
